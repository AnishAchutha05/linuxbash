Linux & Bash
Mastery Roadmap

Everything you need to genuinely understand the Linux command line and bash scripting — built for your path toward GCP and Docker.

6  milestones ~10–14  weeks Ubuntu / any distro basics → advanced

Before you type a single command — understand what Linux actually is, how the terminal works, and why it matters for a DevOps engineer.

hands-on tasks
1
Terminal Orientation
Open a terminal. Run pwd, ls -la, cd /, ls, cd ~ and describe in a note what each output means. Map out the root filesystem mentally.
pwd && ls -la && cd / && ls
2
Read a Manual Page
Run man ls. Navigate with arrow keys. Find the flag that sorts by file size. Exit with q. Then do the same for man cp.
man ls
3
History & Shortcuts
Run 10 different commands. Use Ctrl+R to reverse-search for one of them. Use !! to re-run the last command. List your history with the history command.
history | tail -20

hands-on tasks
1
Build a project folder structure
Using only the terminal, create ~/projects/myapp with subdirs src/, logs/, config/. Create 3 placeholder .txt files inside src/ using touch. Move one to config/.
mkdir -p ~/projects/myapp/{src,logs,config}
2
Read a real log file
Navigate to /var/log. Use tail -50 syslog (or auth.log). Then use tail -f on it — leave it open for 30 seconds and watch entries. Exit with Ctrl+C.
tail -f /var/log/syslog
3
Find all .conf files on the system
Use find to search /etc for all files ending in .conf. Pipe the output to wc -l to count them. Try adding -mtime -7 to find ones modified in the last week.
find /etc -name "*.conf" | wc -l

hands-on tasks
1
Permission Experiment
Create a file, set it to 000. Try to cat it — see the denial. Change to 644, try again. Create a shell script, give it 755 and run it. Understand each permission digit.
chmod 000 test.txt && cat test.txt
2
Find what's eating your CPU
Run htop. Sort by CPU usage (F6 → CPU%). Identify the top 3 processes. Find the PID of one, and use kill -15 PID on a safe test process. Observe the result.
ps aux --sort=-%cpu | head -10
3
Service control with nginx
Install nginx with apt. Start it. Check status. Hit localhost:80 in a browser. Stop it. Re-check status. Look at its logs with journalctl -u nginx --since "10 min ago".
sudo systemctl status nginx

Script Basics

Variables & Input

Conditionals
+

hands-on tasks
1
System health check script
Write a script that prints disk usage, RAM usage, uptime, and logged-in users. Add a warning if disk usage is above 80%. Run it, make it executable, and schedule it as a one-shot test.
#!/usr/bin/env bash · df -h · free -h · uptime
2
Bulk file renamer
Write a script that accepts a directory as $1. It loops over all .txt files in that dir and renames them to .bak. Handle the case where the dir doesn't exist with an error message and exit 1.
for f in "$1"/*.txt; do mv "$f" "${f%.txt}.bak"; done
3
Automated backup script
Write a script that tarballs a given directory ($1), names the archive with a timestamp, saves it to ~/backups/, and logs each backup to ~/backups/backup.log with the date and size.
tar -czf backup_$(date +%Y%m%d_%H%M%S).tar.gz "$1"
milestone 05 / 06
Power Tools
The tools that turn good Linux users into elite ones. grep, awk, sed, and friends — you'll use these every day as a DevOps engineer analyzing logs and data.
intermediate
01
grep — Pattern Matching
+
grep "pattern" file — basic search
-i (case insensitive), -r (recursive), -l (filenames only)
-n (line numbers), -c (count matches)
-v to invert match (show non-matching lines)
-E for extended regex (grep -E "error|warn")
grep in pipelines: command | grep keyword
02
sed — Stream Editor
+
sed 's/old/new/g' — global substitution
-i flag for in-place file editing
Deleting lines matching a pattern: sed '/ERROR/d'
Print specific lines: sed -n '5,10p'
Address ranges and line numbers
03
awk — Data Processing
+
awk '{print $1, $3}' — print specific fields
Field separator: awk -F: '{print $1}' /etc/passwd
Conditions: awk '$3 > 1000 {print $0}'
NR (record num), NF (field count) variables
BEGIN and END blocks
Summing columns: awk '{sum += $2} END {print sum}'
04
sort, uniq, cut
+
sort alphabetically and numerically (-n)
sort -r (reverse), sort -k2 (by column 2)
uniq to deduplicate (must sort first!)
uniq -c to count occurrences
cut -d: -f1 to extract columns from delimited files
05
Networking Commands
+
ping, traceroute, mtr for connectivity
curl and wget for HTTP requests
netstat -tulpn and ss -tulpn for open ports
ip addr, ip route for interfaces
nmap for port scanning (lab use)
SSH basics: ssh user@host, ssh keys
06
Archiving & Compression
+
tar -czf archive.tar.gz dir/ — create
tar -xzf archive.tar.gz — extract
tar -tf to list contents without extracting
gzip, gunzip, bzip2, xz comparison
zip/unzip for cross-platform use
07
Cron & Scheduling
+
crontab -e — editing your cron jobs
Cron syntax: minute hour day month weekday
@reboot, @daily, @weekly shortcuts
Redirect cron output to logs
Debugging cron with journalctl
hands-on tasks
1
Log analysis pipeline
Using /var/log/syslog (or any log file), build a one-liner pipeline: extract only ERROR lines, count occurrences of each unique error type, sort by frequency, show top 5.
grep -i "error" syslog | awk '{print $5}' | sort | uniq -c | sort -rn | head -5
2
Scheduled backup with cron
Take your backup script from Milestone 4. Schedule it with cron to run every day at 2am. Verify crontab -l shows it. Check journalctl to confirm it ran after one cycle.
crontab -e · 0 2 * * * /home/user/backup.sh
3
SSH keygen & passwordless login
Generate an SSH key pair with ssh-keygen. Copy the public key to another machine (or a second local user) with ssh-copy-id. Log in without a password. This is how all DevOps tooling connects to servers.
ssh-keygen -t ed25519 && ssh-copy-id user@host

API caller script with jq

Build your personal .bashrc toolkit
Add 5 useful aliases (e.g., alias gs="git status"), a function that makes a directory and immediately cd's into it (mkcd), and a custom PS1 prompt showing the current git branch. Source it and test everything.

Capstone: Docker-ready deploy script

./deploy.sh — your first real DevOps script
